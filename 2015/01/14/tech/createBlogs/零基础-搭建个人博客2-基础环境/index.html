本篇目的

在通过本篇可以完成博客基础环境的搭建，并能正常在本地运行属于自己的博客基础模板样式。

基础环境：

1.安装nodejs：

https://nodejs.org/zh-cn

点击上述网站，选择适合自己系统的nodejs安装包，一路默认安装配置。
安装完成后在终端执行如下命令：

npm -v

能正常展示版本号，即表明nodejs安装成功；

2.配置nodejs地址仓库：

阿里云：npm config set registry http://registry.npmmirror.com

原始仓库：npm config set registry http://registry.npmjs.org

由于海外地址和资源的限制等问题，导致默认的原始仓库无法链接时；
可以使用国内阿里云的仓库。具体配置方法，在终端或者bash窗口中输入：

npm config set registry http://registry.npmmirror.com

3.安装git：

https://git-scm.com/

点击上述网站，选择适合自己系统的nodejs安装包，一路默认安装配置。
安装完成后在终端执行如下命令：

git -v

能正常展示版本号，即表明git安装成功；

4.安装Hexo：

打开终端或者bash窗口执行：

sudo npm install hexo-cli -g

安装完成后在终端执行如下命令：

hexo -v

能正常展示版本号，即表明hexo安装成功；

5.初始化blog文件夹：

打开终端或者bash窗口，进入到计划您防止代码的文件夹。

然后，执行如下命令进行博客系统的初始化：

hexo init test(这个可以自定义)

test 为创建的目录，你可以自己定义(您的名字或者昵称都可以)

这样便可以直接启动初始化的博客，启动后控制台输出相关信息，我们发现存在问题：

根据提示，我们做出如下调整后，再次启动；

6.初次访问系统：

通过步骤5我们已经完成了博客的初始化工作；

接下来我们通过终端或者bash窗口进入到刚刚创建的test目录中，后续我们称这里为根目录，接下来启动hexo服务：

hexo s
